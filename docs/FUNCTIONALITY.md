# Функциональность PDF Reader

Клиентское SPA для просмотра PDF с комментариями на странице, вложениями и инструментами, близкими к Mozilla PDF.js Viewer. Стек: React, MUI, react-pdf (pdf.js), React Query, MSW (dev), архитектура FSD.

## Маршруты

| Путь                                               | Страница            | Описание                                               |
|----------------------------------------------------|---------------------|--------------------------------------------------------|
| `/`                                                | Список документов   | Карточки доступных PDF                                 |
| `/documents/:documentId`                           | Просмотр документа  | Основной режим чтения                                  |
| `/documents/:documentId/attachments/:attachmentId` | Просмотр вложения   | Прикреплённый PDF в том же UI                          |
| `/documents/:documentId/attachments/embedded`      | Встроенное вложение | PDF из `getAttachments()` pdf.js (состояние навигации) |

Неизвестные пути перенаправляются на `/`.

## Список документов

- Загрузка списка через `GET /api/documents`.
- Каждый документ: `id`, `title`, `pdfUrl`.
- Переход в просмотр по клику на карточку.

## Просмотр PDF

### Загрузка файла

- PDF сначала скачивается в основном потоке (`fetch`), затем передаётся в react-pdf как `{ data: Uint8Array }` — без повторного запроса из worker.
- Относительные `pdfUrl` учитывают `base` Vite (`resolvePdfSrc`).
- Демо-файл: `public/sample.pdf` (скрипт `npm run ensure-sample-pdf`).

### Панель инструментов

- **Боковая панель** — миниатюры, оглавление, вложения.
- **Поиск** — строка поиска, учёт регистра, навигация по совпадениям, подсветка в текстовом слое.
- **Страницы** — первая / предыдущая / следующая / последняя; поле номера страницы (ввод + **Enter** или потеря фокуса).
- **Масштаб** — уменьшить / увеличить (шаг 1.1, как pdf.js); выпадающий список пресетов.
- **Дополнительно** (меню ⋮) — режимы прокрутки, разворот, инструмент курсора, поворот, полноэкранный режим.

### Масштаб (логика pdf.js)

Пресеты в селекторе:

| Значение         | Поведение                                    |
|------------------|----------------------------------------------|
| Авто             | До 125% ширины, с учётом ориентации страницы |
| Реальный размер  | 100% (scale = 1)                             |
| Вписать страницу | Вся страница в видимую область               |
| По ширине        | Ширина страницы = ширина контейнера          |
| 50% … 200%       | Фиксированный числовой масштаб               |
| Произвольный %   | После кнопок +/- (например 110%, 121%)       |

Пересчёт при изменении размера окна, поворота, режима прокрутки.

### Режимы прокрутки

- **Постранично** — виден текущий разворот (с учётом spread).
- **Вертикальная** — все страницы столбцом, синхронизация номера страницы при прокрутке.
- **Горизонтальная** — страницы в ряд.
- **Плитка** — перенос строки страниц.

### Разворот (spread)

- Одна страница.
- Разворот нечётных (первая отдельно, далее пары 2–3, 4–5…).
- Разворот чётных (смещение для выравнивания).

### Инструмент «Рука» / «Выделение»

- **Выделение** — текстовый слой включён, можно выделять текст PDF, прокрутка колёсиком.
- **Рука** — перетаскивание области просмотра (pan), текстовый слой отключён.

### Поворот

- По часовой / против часовой на 90° (накопительный угол 0° / 90° / 180° / 270°).

### Полноэкранный режим

- Fullscreen API на колонку просмотра PDF.
- Скрываются заголовок, тулбар, поиск, панель инструментов комментариев.
- Документ на весь экран, тёмно-серый фон как в pdf.js.
- Выход: Esc или повторный пункт меню.

### Поиск по документу

- Панель: запрос, «Учитывать регистр», счётчик «N из M», предыдущее / следующее совпадение.
- Поиск по `getTextContent()` pdf.js, подсветка через `customTextRenderer` в текстовом слое (светло-синий фон, активное совпадение ярче).
- При первом результате — переход на страницу.
- Закрытие панели сбрасывает подсветку и запрос.

### Боковая панель (слева)

**Миниатюры** — превью страниц, клик переходит на страницу; текущая выделена рамкой.

**Оглавление** — дерево из PDF outline; клик по пункту с известной страницей — переход.

**Вложения** — см. раздел «Вложения к документу».

На узких экранах панель — оверлей с затемнением; закрытие по клику вне.

### Горячие клавиши

| Комбинация    | Действие            |
|---------------|---------------------|
| Ctrl/Cmd + F  | Открыть поиск       |
| Ctrl/Cmd + +  | Увеличить масштаб   |
| Ctrl/Cmd + -  | Уменьшить масштаб   |
| Page Down / ↓ | Следующая страница  |
| Page Up / ↑   | Предыдущая страница |
| Home          | Первая страница     |
| End           | Последняя страница  |

## Комментарии

### Панель комментариев (справа)

- Список **потоков** (корневой комментарий + ответы).
- Чекбокс **«Решён»** у корневого — помечает ветку решённой (ответы тоже).
- Ответы в стиле Instagram (один уровень вложенности).
- Удаление корневого удаляет всю ветку.
- Клик по комментарию — выделение и переход на страницу якоря.

### Инструменты на странице

| Инструмент | Действие                                            |
|------------|-----------------------------------------------------|
| Просмотр   | Только просмотр и выбор существующих меток          |
| Текст      | Выделение текста на странице → черновик комментария |
| Подсветка  | Прямоугольник мышью → черновик с областью           |
| Метка      | Клик по странице → якорная точка                    |

### Создание комментария

- После выбора области открывается **inline-редактор** (Portal поверх страницы).
- Выбор **цвета подсветки** (палитра, сохранение в `localStorage`).
- Сохранение / отмена.
- Подсветки на странице: прямоугольники в нормализованных координатах (0…1).
- Решённые комментарии — приглушённая зелёная подсветка.

### Координаты

- `pageIndex` — с нуля.
- `relX`, `relY` — доля ширины/высоты страницы.
- `highlightRects` — опционально, массив нормализованных прямоугольников.

## Вложения к документу

### Прикреплённые (пользовательские)

- Кнопка **«Прикрепить PDF»** во вкладке «Вложения».
- Только PDF, до **25 МБ**.
- Хранение в MSW в памяти до перезагрузки страницы.
- **Открыть в режиме чтения** — тот же `PdfWorkspace`, заголовок «Вложение к документу …», кнопка «← К документу».
- **Удалить** — только прикреплённые.

### Встроенные в PDF

- Извлекаются через `pdf.getAttachments()` при загрузке документа.
- Секция «Встроенные в PDF», открытие в режиме чтения (blob URL).
- Удалить нельзя.

## Адаптивность

- На `lg+`: комментарии справа, боковая панель PDF слева в потоке.
- Ниже `lg`: колонка комментариев под просмотром; боковая панель PDF — оверлей.

## REST API (контракт)

База: `VITE_API_BASE_URL` (по умолчанию `/api`).

### Документы

| Метод | Путь                     | Описание   |
|-------|--------------------------|------------|
| GET   | `/documents`             | Список     |
| GET   | `/documents/:documentId` | Метаданные |

### Комментарии

| Метод  | Путь                              | Описание                      |
|--------|-----------------------------------|-------------------------------|
| GET    | `/documents/:documentId/comments` | Список                        |
| POST   | `/documents/:documentId/comments` | Создать / ответить            |
| PATCH  | `/comments/:commentId`            | Текст или `resolved`          |
| DELETE | `/comments/:commentId`            | Удалить (ветку для корневого) |

Заголовки автора (опционально): `X-User-Id`, `X-User-Name`.

### Вложения

| Метод  | Путь                                                    | Описание                                      |
|--------|---------------------------------------------------------|-----------------------------------------------|
| GET    | `/documents/:documentId/attachments`                    | Список метаданных                             |
| POST   | `/documents/:documentId/attachments`                    | Загрузка (`multipart/form-data`, поле `file`) |
| GET    | `/documents/:documentId/attachments/:attachmentId/file` | Байты файла                                   |
| DELETE | `/documents/:documentId/attachments/:attachmentId`      | Удалить                                       |

## Моки (разработка)

- MSW перехватывает `/api/*` в `import.meta.env.DEV`.
- `documents.json`, `comments.json` — начальные данные.
- Вложения и новые комментарии — в памяти до F5.
- `GET */sample.pdf` — passthrough к статике Vite.

## Структура кода (FSD)

| Слой                             | Ключевые модули                                          |
|----------------------------------|----------------------------------------------------------|
| `app/`                           | Роутинг, провайдеры, MSW, глобальные стили               |
| `pages/`                         | `HomePage`, `DocumentReaderPage`, `AttachmentReaderPage` |
| `widgets/`                       | `PdfWorkspace`                                           |
| `features/pdf-viewer/`           | Тулбар, поиск, сайдбар, масштаб, меню                    |
| `features/manage-comments/`      | Инструменты, панель, inline-редактор, подсветки          |
| `features/document-attachments/` | Вкладка вложений                                         |
| `entities/document/`             | Документ, запросы                                        |
| `entities/comment/`              | Комментарии, мутации                                     |
| `entities/document-attachment/`  | API вложений                                             |
| `shared/`                        | REST-клиент, PDF worker, утилиты, моки                   |

## Переменные окружения

- `VITE_API_BASE_URL` — базовый URL API (по умолчанию `/api`).
